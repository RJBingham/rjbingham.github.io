Homodimers

A short guide on how to calculate structural similarity between two polypeptide chains.

Calculating RMSD using PyMol (two molecules)

If you have two separate PDB files they will appear as two lines with separate controls. In the image below I have loaded in structures for 7JTL and 7MX9. One way to superimpose them is to click on the A for ‘Action’, then select ‘align’, ‘to molecule’, then select the template, in this case 7JTL. The output in the console will report the RMSD and the number of C-alpha atoms (CA).

 Executive: RMSD =    0.410 (71 to 71 atoms)
 Executive: object "aln_7MX9_to_7JTL" created.

To visualise the result, you will need to re-centre on the molecule, use the Action-Orient or Action-Center options to bring the molecules back into the middle of the screen.

Calculating RMSD using PyMol (two chains within the same PDB file)

If you have a single PDB file with two chains, the above procedure will not work. You need to define the two objects that need to be superimposed. We will use the dimer in 7JTL as an example. Notice that there are two polypeptide chains. Right-click on each to see the Chain ID. In this case, the chains are A and B.

Use the commands below to create separate objects for each chain, then superimpose them based on the CA atoms. These commands need to be entered into the command prompt in PyMOL.

create chainA, 7JTL and chain A 
create chainB, 7JTL and chain B 
align chainA and name CA, chainB and name CA

The output will appear as below:

PyMOL>align chainA and name CA, chainB and name CA
 Match: read scoring matrix.
 Match: assigning 107 x 107 pairwise scores.
 MatchAlign: aligning residues (107 vs 107)...
 MatchAlign: score 585.000
 ExecutiveAlign: 107 atoms aligned.
 ExecutiveRMS: 5 atoms rejected during cycle 1 (RMSD=2.43).
 ExecutiveRMS: 5 atoms rejected during cycle 2 (RMSD=0.99).
 ExecutiveRMS: 5 atoms rejected during cycle 3 (RMSD=0.45).
 ExecutiveRMS: 3 atoms rejected during cycle 4 (RMSD=0.29).
 Executive: RMSD =    0.260 (82 to 82 atoms)

Now try ‘super’ instead of ‘align’. Notice that the RMSD is different, and fewer atoms were chosen.

 PyMOL>super chainA and name CA, chainB and name CA
 MatchAlign: aligning residues (107 vs 107)...
 MatchAlign: score 434.622
 ExecutiveAlign: 97 atoms aligned.
 ExecutiveRMS: 6 atoms rejected during cycle 1 (RMSD=2.79).
 ExecutiveRMS: 4 atoms rejected during cycle 2 (RMSD=1.22).
 ExecutiveRMS: 4 atoms rejected during cycle 3 (RMSD=0.50).
 ExecutiveRMS: 4 atoms rejected during cycle 4 (RMSD=0.30).
 ExecutiveRMS: 1 atoms rejected during cycle 5 (RMSD=0.26).
 Executive: RMSD =    0.252 (78 to 78 atoms)

‘align’ performs a sequence-dependent structural alignment. This should only be used when the proteins are homologous.

‘super’ performs a sequence-independent, structure-based alignment. This is more tolerant of insertions/deletions and structural differences. Best when proteins are similar in fold but not necessarily in sequence

TM-Score (Zhang Lab)

the TM-score is another method to calculate structural similarity. It overcomes some of the limitations of the RMSD score. You can calculate a TM score and read about the advantages of this method here https://aideepmed.com/TM-score/
